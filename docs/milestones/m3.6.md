# M3.6 — User Onboarding

> **Goal**: Help new users set up their account after registration. Onboarding is minimal, functional, and transparent — not manipulative. No algorithmic profiling.

---

## Philosophy

YoTop10 is deliberately **anti-algorithm**. We don't track user behavior to personalize feeds. Onboarding exists purely to:

1. Help users customize their experience
2. Let them choose what content categories to follow (functional filtering)
3. Set privacy preferences

Onboarding should feel like **setting up preferences**, not being **profiled**.

---

## What Onboarding Covers

| Step | Purpose | Required |
|------|---------|----------|
| Display Name | Personalize your profile | No |
| Avatar | Profile picture | No |
| Category Selection | Set default category filter | No |
| Privacy Preference | Default post visibility | No |
| Theme Selection | Retro or Futuristic | No |

---

## User Flow

```
Registration Complete
        ↓
[Optional] Prompt: "Set up your profile"
        ↓
Step 1: Display Name + Avatar (optional)
        ↓
Step 2: Pick Categories (optional)
        ↓
Step 3: Default Privacy (optional)
        ↓
Step 4: Theme (optional)
        ↓
Main Feed (newest posts, filterable by category)
```

---

## Implementation Notes

### Category Selection

- Show top 10 categories as selectable chips
- User can select 0-5 categories
- Selected categories become the default filter on home feed
- Users can always change this in settings
- **This is NOT algorithmic personalization** — it's explicit filtering

### Privacy Preference

- Public: Anyone can see your posts
- Connections Only: Only approved connections see your posts
- Private: Only you see your posts

### Skip Option

- Every step has a "Skip" or "Later" option
- Users can complete onboarding anytime from Settings
